# Keywords for SetAttr META

The following keywords can be used to modify file metadata [programmatically](/Manual/file_operations/editing_metadata/programmatic_setting_of_metadata.md) with the **[SetAttr](../command_reference/internal_commands/setattr.md) META** command. Most fields can only be set to a string (or number), or cleared, however some fields accept more complex instructions, and these are noted below.

| Field | Keyword | Notes |
| --- | --- | --- |
| **Standard Properties** |  |  |
| Attributes | attr | Attributes are specified with one or more of the following letters:  <br />**A** (archive)  <br />**R** (read-only)  <br />**H** (hidden)  <br />**S** (system)  <br />**C** (compressed)  <br />**E** (encrypted)<br /><br />You can also use **+** to turn attributes on and **-** to turn attributes off.<br /><br />//<Example://><br /><br />**SetAttr META attr:+c-r***   - sets compression and clears read-only attributes* |
| Date created | createdate | The date and time can be set to an absolute date (in which case the time will be unchanged), an absolute time (in which case the date will be unchanged), or both.  <br />The accepted formats for an absolute time and date are:  <br />**YYYY-MM-DD** - set just the date  <br />**HH:MM:SS** - sets just the time  <br />**YYYY-MM-DD HH:MM:SS** - sets both time and date (needs quotes)  <br />The keyword **now** can also be given to use the current date and time.  <br />You can also perform relative adjustments to the current date and time setting, using the following formats:  <br />**\<time-adjust\> ** - adjusts only the time  <br />**\<date-adjust\> \<time-adjust\>**  - adjusts both date and time (needs quotes)<br /><br />The valid formats of the **\<time-adjust\>** string are:  <br />**\[+-\]H:M:S**  - add or subtract hours, minutes and seconds  <br />**\[+-\]H:M**  - add or subtract hours and minutes  <br />**\[+-\]H**  - add or subtract a number of hours  <br />The valid formats of the **\<date-adjust\>** string are:  <br />**\[+-\]Y:M:D**  - add or subtract a year, month and day value  <br />**\[+-\]M:D**  - add or subtract a month and day value  <br />**\[+-\]D**  - add or subtract a number of days<br /><br />You can't adjust the date without also adjusting the time - so specify 0 for *\<time-adjust\>* if you only want to adjust the date.<br /><br />*Example*:<br /><br />**SetAttr META createdate:now***   - sets the creation date to the current date/time*  <br />**SetAttr META createdate:20100922***   - sets creation date to Sep 22, 2010*  <br />**SetAttr META "createdate:20100922 15:30:30"***   - also sets time to 3:30 pm*  <br />**SetAttr META createdate:+1:30***   - adds 1h30m to creation time*  <br />**SetAttr META "createdate:+1 0"***   - adds 1 day (and zero hours)*<br /><br />You can also copy the values from another date field by specifying the other field's name: **createdate**, **lastmodifieddate**, **datedigitized** or **datetaken**. (For backward compatibility, **modifydate** also works as an alias of lastmodifieddate.)<br /><br />*Example*:<br /><br />**SetAttr META createdate:lastmodifieddate***   - sets the creation date to the file's last modified date/time* |
| Date modified | lastmodifieddate | Accepts the same values as **createdate** (described above). |

| Tags | tags | Accepts multiple semi-colon separated tags. You can either set the tags absolutely, or add tags to or remove tags from the existing set.<br /><br />*Example*:<br /><br />**SetAttr META tags:one;two***   - sets the tags to "one" and "two"*  <br />**SetAttr META tags:+one***   - adds the tag "one" to any existing tags*  <br />**SetAttr META tags:+one;-two***   - adds "one" and removes "two"* |
